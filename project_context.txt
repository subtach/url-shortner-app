=============================================
  SHORTLY — URL SHORTENER & SHARING PLATFORM
  Project Context for PPT
=============================================

PROJECT OVERVIEW
----------------
Shortly is a full-stack web application that lets users shorten long URLs,
share code snippets, and share images — all via short, memorable links
with optional QR codes.


PROBLEM STATEMENT
-----------------
Long URLs are hard to share, remember, and track. Developers also need a
quick way to share code snippets and images without heavy third-party
platforms. Shortly solves all three in one place.


KEY FEATURES
------------
1. URL Shortener     - Convert any long URL into a short link (e.g. shortly.app/abc123)
2. Custom Aliases    - Users can set their own alias (e.g. shortly.app/my-project)
3. Click Tracking    - Real-time click count per link
4. Link Expiry       - Set an expiry date; expired links show a friendly message
5. Code Paste        - Share code snippets with syntax highlighting via short link
6. Image Sharing     - Upload and share images (JPEG, PNG, GIF, WebP, SVG, AVIF)
7. QR Code Generator - Every short link gets a downloadable QR code (SVG format)
8. Dashboard         - Manage all links — view, copy, and delete


TECH STACK
----------
Framework   : Next.js 16 (React frontend + Node.js backend in one)
Language    : TypeScript
Styling     : Tailwind CSS
Database    : Turso (cloud-hosted SQLite via libSQL)
File Storage: Netlify Blobs (serverless image storage)
Deployment  : Netlify (with GitHub CI/CD)
QR Codes    : qrcode npm library (SVG output, works in serverless)


ARCHITECTURE
------------
User Browser
    |
    v
Next.js App (hosted on Netlify Functions)
    |
    |-- /               --> URL Shortener UI
    |-- /paste          --> Code Paste UI
    |-- /media          --> Image Upload UI
    |-- /dashboard      --> Link Management
    |-- /[shortCode]    --> Redirect Handler
    |-- /paste/[code]   --> Paste Viewer
    |-- /media/[code]   --> Image Viewer
    |
    |-- Turso DB        --> Stores metadata (links, pastes, media info)
    |-- Netlify Blobs   --> Stores actual uploaded image files


HOW IT WORKS (URL Shortening - Step by Step)
--------------------------------------------
1. User enters a long URL (+ optional custom alias & expiry date)
2. Server generates a 6-character random short code (or uses custom alias)
3. Record is saved to Turso cloud database
4. User instantly gets a short URL + QR code
5. Anyone visiting the short URL is automatically redirected to original URL


DEPLOYMENT PIPELINE
-------------------
Developer pushes code --> GitHub repo --> Netlify auto-deploys
- Serverless API routes are handled by Netlify Functions
- Database is hosted on Turso (global edge SQLite, low-latency)
- Images stored in Netlify Blobs (serverless object storage)

HIGHLIGHTS TO MENTION
---------------------
- 3-in-1 platform: URLs + Code Snippets + Images
- Serverless architecture — scales automatically, no server to manage
- Real-time click analytics on every link
- QR code generation for every short link
- Cloud database (Turso) with global low-latency access
- Clean, responsive UI — works on both mobile and desktop
- Automatic deployment pipeline via GitHub and Netlify
